Test-Driven Development (TDD) is a software development methodology and an agile technique that requires a failing test to be written before code, and then writing the minimal code that would pass the test, and then refactor the code continously.

In TDD, you will end up having at least one test for every single feature/functionality in the system which increases the reliablity of the system. All these tests should be run atumaticaly using some testing frameworks.

TDD forces programmers to think throughly about the feautre they need to implement before actually implementing it. Also, it forces them to write the minmal amount of code that would implement the feauture. Coding only the required feature resutls in having a code that is simple and clean and hence, easy to change and refactor.
